# DockStack Privacy Policy

_Last updated: 2026-07-02_

This privacy policy describes how DockStack handles data when you use the browser extension, its local native core, and associated local processing features.

## Summary

DockStack is designed to operate as a **local-first tool**.

By default, DockStack is built to:
- capture browsing-related technical data only when the user starts a capture session
- store that data locally on the user’s own machine
- avoid remote transmission of captured data unless the user explicitly chooses an action that depends on an external or local network endpoint

## What DockStack can access

Depending on the browser permissions granted and the features the user enables, DockStack may process:
- tab URLs
- page URLs
- request URLs
- request methods
- request and response headers
- request payloads
- response payloads
- content types
- local settings and session metadata

If the user enables **sensitive capture mode**, DockStack may also encounter authenticated or session-linked traffic.

## Sensitive mode

Sensitive capture mode is disabled by default.

When sensitive mode is enabled by the user:
- DockStack may inspect request/response metadata associated with authenticated application flows
- DockStack may encounter secret-bearing values such as authorization or CSRF-related headers
- DockStack applies masking rules before persistence for certain known sensitive header names

Sensitive mode requires explicit user acknowledgement inside the extension UI.

## Local storage

DockStack currently uses local storage mechanisms such as:
- browser extension local storage
- SQLite
- DuckDB
- local export files generated by user action

This data remains on the user’s own system unless the user intentionally moves, shares, or uploads it.

## Local AI integrations

DockStack may integrate with locally hosted AI runtimes such as Ollama or llama.cpp-compatible services.

If the user invokes an analysis feature:
- selected context may be sent to a local endpoint such as `http://127.0.0.1`
- this is intended to remain on the user’s own machine or controlled local environment

DockStack does not require a paid cloud AI API to function.

## Data sharing

DockStack does not advertise itself as a cloud synchronization service in this repository.

The default expected behavior is:
- no background upload of captured data to a vendor-controlled server
- no hidden telemetry pipeline for captured request bodies
- no silent external export of session artifacts

## User-controlled export

DockStack allows the user to export captured data and generated outputs.

These exports are user-triggered operations and may include:
- JSON
- CSV
- packaged local artifacts

Users are responsible for how they store, share, or redistribute exported material.

## Browser permissions rationale

DockStack requests permissions required for its capture and analysis workflow. These permissions are intended to support:
- capture controls
- content injection
- local downloads
- native messaging
- tab context and page interaction

A detailed permission-by-permission explanation should be provided alongside store listings and release documentation.

## Retention

Captured data may remain available locally until:
- the user clears browser extension data
- the user deletes local storage files
- future retention tooling removes old sessions

Retention behavior can change as DockStack evolves.

## Security limitations

No client-side tool can guarantee zero risk when handling sensitive browser traffic.

Users should treat DockStack as a powerful local instrument and only use it in environments where they are authorized to inspect the data being captured.
